Система обліку супермаркету

Веб-застосунок для автоматизації торговельних та складських процесів супермаркету.

---

Як запустити проєкт

Вимоги
- [Docker](https://www.docker.com/get-started) та Docker Compose

Запуск
```bash
git clone <посилання на репозиторій>
cd supermarket_accounting
docker compose up --build
```

або завантажте файли і
```bash
docker compose up --build
```

Після запуску відкрийте браузер і перейдіть за адресою:
```
http://localhost:5173
```

---

Які порти використовуються

| Сервіс    | Порт  | Опис                        |
|-----------|-------|-----------------------------|
| Frontend  | 5173  | Веб-інтерфейс (React + Vite)|
| Backend   | 3000  | REST API (Node.js + Express)|
| MongoDB   | 27017 | База даних                  |

---

Як користуватись застосунком

| Вкладка   | Опис                                                 |
|-----------|------------------------------------------------------|
| Панель    | Загальний огляд: кількість товарів, замовлень, виторг|
| Товари    | Каталог товарів, додавання та редагування            |
| Склад     | Залишки на складі, терміни придатності               |
| Продажі   | Оформлення нових замовлень, історія продажів         |
| Персонал  | Список співробітників, їх ролі та статус             |
| Постачання| Список постачальників та контракти                   |
| Звіти     | Звіт по товарах із терміном придатності що спливає   |

---

Міграції бази даних

Міграції виконуються автоматично при першому запуску — додаткових дій не потрібно.

При першому запуску система автоматично створює всі колекції та наповнює базу демонстративними даними

Якщо потрібно скинути базу і почати з нуля:
```bash
docker compose down -v
docker compose up --build
```

Якщо потрібно зупинити і продовжити (дані зберігаються):
```bash
docker compose stop
docker compose start
```

Технічний стек

- Frontend: React, Vite
- Backend: Node.js, Express.js
- База даних: MongoDB 7.0
- Контейнеризація: Docker, Docker Compose
